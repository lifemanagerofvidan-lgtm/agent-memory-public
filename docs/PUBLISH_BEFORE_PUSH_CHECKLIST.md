# Publish Before Push Checklist

Use this checklist before the first GitHub push.

## Public safety

- [ ] No secrets or tokens are present
- [ ] No private notes or unrelated workspace files are present
- [ ] No accidental screenshots, temp files, or caches are present
- [ ] Local paths that remain are either intentional defaults or clearly documented examples

## Repo completeness

- [ ] `README.md` explains what the project is
- [ ] Setup instructions exist
- [ ] Validation instructions exist
- [ ] Troubleshooting instructions exist
- [ ] Examples exist
- [ ] `.gitignore` exists

## GitHub readiness

- [ ] Repo name is final enough
- [ ] Visibility choice is intentional, preferably private first
- [ ] Initial commit message is ready
- [ ] GitHub CLI is logged in if using `gh`

## Final human check

- [ ] I would be comfortable with a stranger reading this repo today
- [ ] The first screen of the README feels clear and credible
- [ ] The repo reflects what I actually want to publish
